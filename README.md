# Конструктор сценариев

Прототип модуля для CRM — список сценариев + визуальный редактор блок-схем.

## Запуск

```bash
npm install
npm run dev
```

Приложение откроется на `http://localhost:5173`.

Для продакшн-сборки:

```bash
npm run build
npm run preview
```

## Стек

- **React 19** + **TypeScript** — строгая типизация, функциональные компоненты с хуками
- **Vite** — сборка и дев-сервер
- **React Flow (@xyflow/react)** — отрисовка графа, drag-n-drop, соединение узлов, удаление
- **React Router** — навигация между списком и редактором
- **CSS Modules** — стилизация без конфликтов имён
- **localStorage** — хранение данных (бэкенд имитируется через async-обёртки с задержкой)

## Архитектура

```
src/
├── api/              # Имитация API (localStorage + задержка 300–400мс)
├── components/
│   ├── FlowEditor/   # Редактор схемы: холст, узлы, snap-линии
│   ├── ScenarioList/ # Страница списка сценариев
│   └── UI/           # Атомарные компоненты (Button, Spinner)
├── hooks/
│   ├── useAsync.ts      # Универсальный хук loading/error для промисов
│   └── useFlowEditor.ts # Вся бизнес-логика редактора
├── utils/
│   ├── geometry.ts       # Математика (расстояние точки до отрезка)
│   ├── getHelperLines.ts # Алгоритм snap-выравнивания
│   └── reconnectEdges.ts # Пересоединение цепочки при удалении узла
├── types/            # Интерфейсы: ScenarioNode, ScenarioEdge, Scenario, NodeData
├── App.tsx
└── main.tsx
```

**Основные решения:**

- Бизнес-логика (CRUD, асинхронность) изолирована в `api/` — компоненты её не знают.
- Вся логика редактора (загрузка, сохранение, snap, вставка в ребро, пересоединение) вынесена в `useFlowEditor`. Компонент `FlowEditor` — чистый UI без логики.
- Утилиты без UI живут в `utils/`, а не в `components/` — направление зависимостей однонаправленное.
- Типы узлов регистрируются через фабрику `makeNodeComponent(variant)` — нет двух файлов-однострочников.
- При удалении узла с существующими связями цепочка сохраняется: A→B→C при удалении B превращается в A→C.

## Использование ИИ

При работе над проектом использовал Claude для нескольких вещей:

- **Бойлерплейт**: попросил накидать стартовую структуру компонентов, чтобы не тратить время на рутинное создание файлов. Потом переписал под свою архитектуру.
- **CSS**: часть стилей (спиннер, кнопки) генерировал с помощью ИИ и подправлял под общий стиль.
- **Типы для React Flow**: спрашивал по API библиотеки `@xyflow/react`, потому что с ней раньше не работал. Проверял по документации на сайте, так же попросил провести мне краткий инструктаж по возможностям библиотеки и подсказки когда собирался решить какую либо задачу
- **Структура** просил првоерить структуру на косяки ,где я отошел от принципа kiss / dry / solid

Весь остальной код (логика, роутинг, хуки, API-слой) писал сам. Каждый сгенерированный фрагмент проверял: смотрел, что типы корректны, запускал, тестировал руками в браузере.
